03
Four Core Innovations

效率革命的四台发动机

3.1  Hybrid Attention · CSA + HCA 交错堆叠

整篇报告最重要的一章。V4 设计了两种互补的注意力机制，然后按层交错使用——不是简单选其一。

CSA · Compressed Sparse Attention "先压缩、再稀疏"

  • 压缩阶段：每 m=4 个 token 的 KV 被可学习的位置偏置 + softmax 权重压缩成 1 条 "KV 摘要"，带前后重叠的对偶键值流 Cₐ/Cᵦ。
  • 稀疏阶段：在压缩序列上套 DeepSeek Sparse Attention——独立的 Lightning Indexer（低秩查询 + ReLU 相似度 + top-k）挑出最相关的块做稠密注意力。
  • 滑动窗口补丁：n_win=128 的 SWA 分支保证局部依赖不被破坏。
  • Attention Sink：每个 head 可学习的 sink logit，允许"主动放弃注意"，总注意力和 < 1。

HCA · Heavily Compressed Attention "极端压缩 + 稠密注意力"

CSA 的 m=4 属于温和压缩；HCA 用的是 m′=128——把 128 个 token 压成 1 条。既然压得这么狠，就不再做稀疏，而是对极短序列做完整稠密注意力。结构简单（无 indexer、无 top-k），但压缩比惊人。

细看近处的 CSA ＋ 远看全局的 HCA — 每层都能扫一眼整个上下文

三级精度策略

RoPE 维度 ……… BF16 （位置信息敏感） KV 其他维度 …… FP8 （KV cache 直接减半） Indexer QK ……… FP4 （top-k 不需精确）

3.2  mHC · Manifold-Constrained Hyper-Connections

Hyper-Connection 把 residual stream 从 R^d 扩展到 R^(n_hc × d)。问题在于堆深就容易数值爆炸。V4 的创新是把 residual mapping 矩阵 B_l 限制到双随机矩阵（Birkhoff 多胞形）流形上：

B_l ∈ M := { M ∈ R^{n×n} | M·1_n = 1_n, 1_n^T·M = 1_n^T, M ≥ 0 }
  • ‖B_l‖₂ ≤ 1 → 映射非扩张，前/反向都不放大信号。
  • 集合 M 在乘法下闭合 → 深堆叠时稳定性可复合。
  • Sinkhorn-Knopp 迭代 20 次投影到流形（先 exp 再交替行/列归一化）。
  • A_l、C_l 用 Sigmoid 限幅 + 非负约束，避免信号抵消。
读者价值

WHY THIS MATTERS

mHC 给想训 100+ 层超深模型的人，提供了一个"理论上有保障、实操上可用"的 residual 升级方案。工程代价：激活内存 + 流水线通信增加，团队用"融合 kernel + 细粒度 tensor-level 重算 + DualPipe 1F1B overlap"把额外时钟压到 6.7%。

3.3  Muon 优化器 · 大规模替换 AdamW

V4 把绝大多数参数从 AdamW 换成 Muon，仅保留 AdamW 用于 embedding、LM head、RMSNorm、mHC 静态偏置与门控。

累积动量 M_t
Nesterov 修正 + 混合 Newton–Schulz 迭代
近似正交化（奇异值拉到 1）
按 √max(n,m)·γ 重标定更新 RMS
与 AdamW 共用超参 · 权重衰减 + 更新

前 8 步用激进系数 (3.4445, −4.7750, 2.0315) 快速收敛；后 2 步切到温和 (2, −1.5, 0.5) 把奇异值稳准地钉在 1。既快又稳。

ZeRO 冲突怎么办

Muon 必须用完整梯度矩阵。用背包算法分桶 + MoE 参数扁平化 + BF16 随机舍入梯度同步 + all-to-all + 本地 FP32 累加：额外显存 <10%，通信量减半。

3.4  DeepSeekMoE 延续 · Hash Routing 开头几层

  • Affinity 打分从 Sigmoid 换成 Sqrt(Softplus(·))——实践证明更稳。
  • 取消 routing target nodes 数量约束，重新设计并行策略补偿。
  • 开头 3 层 MoE 用 Hash Routing——按 token ID 的 hash 直接决定专家，避免早期浅层路由未学好时的不稳定。
  • MTP 深度 1，沿用 V3 的多 token 预测。
04
Infrastructure

让百万上下文真的跑起来

报告里基建的份量和架构几乎一样重。这是 DeepSeek 把自己沉淀的"模型 × 工程"能力向外展示的总汇。

MoE 通信-计算细粒度重叠

一次 MoE 层拆成 3 个 expert wave 流水，通信藏在前一 wave 计算后面。理论加速 1.92×（Comet 1.42×），同时允许互联带宽更低。

TileLang

自研领域专用语言，平衡开发效率与运行效率，替代手写 CUDA。

Batch-Invariant 确定性 kernel 库

保证训练 / 推理 / RL rollout bitwise 可复现——这是 on-policy 算法的命脉。

FP4 量化感知训练

MoE 专家权重和 indexer QK 都用 FP4。RL rollout 直接用真实 FP4 权重，训/部署行为一致。

Tensor-Level Checkpointing

标注几个张量就能自动重算。TorchFX 追踪计算图，反向遍历找到最小重算子图，零多余开销。

Contextual Parallelism 两阶段

① rank 把最后 m 个未压缩 KV 发给下一 rank → ② 合并压缩 → all-gather → 融合 select-and-pad kernel 重组。

异构 KV Cache × 磁盘存储

Full Caching / Periodic Checkpointing / Zero Caching 三策略自由切换。shared-prefix reuse（多轮对话、agent 共享 prompt）成本可降一个数量级——这是 1M 上下文可商用的关键。

DSec 生产级 Agentic 沙箱

Rust 实现，四种基质（Function Call / Container / microVM / fullVM）共用 Python SDK，单集群管数十万并发沙箱，抢占后从 trajectory log 断点续跑。

05
Pre-training & Stability

两个极其实用的稳定性秘方

数据与训练配置

  • 32T+ tokens 多元高质量语料：过滤批量模板化内容、显式加权长文档、扩大多语言、中期加入 agentic data。
  • 序列长度逐步扩：4K → 16K → 64K → 1M；前 1T tokens 用 dense attention warmup，之后在 64K 引入稀疏。
  • Batch peak：V4-Flash 75.5M / V4-Pro 94.4M tokens。峰值 LR：2.7e-4 / 2.0e-4，余弦衰减到 1/10。
  • Auxiliary-loss-free 平衡 + 小权重 sequence balance loss；MTP 权重 0.3，衰减期 0.1。

Loss Spike 的两招

团队坦承训练万亿 MoE 时频繁遇到 loss spike。根因是 MoE 层的 outlier + 路由机制互相加剧。

Anticipatory Routing · 前瞻路由

把特征计算和路由决策解耦：step t 用 θ_t 做特征，但路由索引用 θ_{t-Δt}（提前一步预计算）。异步化打破"特征→outlier→路由→更多 outlier"的恶性循环。

自动检测机制：只在 spike 发生时启用，稳定后自动关闭 → 整体额外开销可忽略。

SwiGLU Clamping

linear 分量限制 [-10, 10]，gate 分量上限 10。简单但有效，直接消除 outlier。

可贵

坦诚 · CANDOR

V4 对这两招的机理坦率写道 "it works but we don't fully understand why"——这种态度在当前"包装过度"的技术报告里尤其稀缺。

Post-training Paradigm

Specialist × On-Policy Distillation

V4 最大的方法论转变：用 OPD 代替混合 RL。

Base Model
SFT 领域高质量数据
GRPO RL · 数学 / 编程 / agent / 指令跟随 · 各一个 expert
On-Policy Distillation 多教师 reverse-KL
Unified Final Model
  • 真 · on-policy，不会蒸馏偏移
  • 避免传统 RL merge 多任务时互相干扰
  • 训练过程共享 KV cache、批量推理，大幅节省算力

Generative Reward Model · GRM

V4 彻底抛弃标量 reward model，让 actor 网络自己充当 GRM——rubric-guided 数据、直接对 GRM 做 RL、评估与生成联合训练。人标数据用量大幅下降。

更多后训练细节

  • Tool-Call · XML + |DSML| 特殊 token：替代 JSON，降低转义错误率，与 think 配合。
  • Interleaved Thinking：工具调用场景完整保留所有思考历史（跨用户消息边界）——1M 窗口真正能被 agent 利用的关键。
  • Quick Instruction：搜索触发 / 意图识别等辅助任务用特殊 token 拼到末端，复用现有 KV cache 并行执行，降低 TTFT。
  • WAL + 持久化 KV cache：rollout 被抢占后从断点恢复——"从头再生成会引入长度偏差"，短回复更易保留，不恢复会让模型被偏向输出短回复。

Four Deep Takeaways

四个深层判断

① 从"参数效率"到"系统效率"

V3 的关键词是 参数激活量（MoE + MLA）；V4 的关键词是 上下文能效——每 token 的 KV 体积 / FLOPs。长 CoT / 长工具轨迹 / 长文档分析把计算从"参数侧"推向了"序列侧"，V4 是对这个转变的正面回应。

② 与前沿闭源的差距仍在 3-6 个月，但性质变了

在硬推理（HMMT、IMO、Codeforces、LiveCodeBench），V4-Pro-Max 与 Gemini-3.1-Pro、GPT-5.4 基本持平甚至领先；在知识类（SimpleQA、MMLU-Pro）仍有较大差距。

差距不再是"推理能力"问题，而是"数据规模 × 知识广度"问题

③ On-Policy Distillation 可能是新主流

分别训 specialist → student 在自己 trajectory 上学 teacher 分布的 reverse KL → 配合 GRM / batch-invariant kernel / 异步 teacher scheduling——这是一套标准化、可规模化的后训练流水线。

④ DeepSeek 的"基建叙事"同样重要

报告第 3 章与第 5.2 节占了近 20 页——TileLang DSL / FP4 QAT / 3FS / DSec / Tensor-level checkpointing——几乎就是"头部基础模型公司这套 stack 都要有"的全景示范。架构创新需要这些工程能力才能跑在 10 万卡 × 万亿参数 × 百万 token 上。

09
Limitations

作者自陈的短板

  1. 架构复杂度偏高：为降风险保留了大量"已验证但非最小必要"组件，未来做 principled distillation 简化。
  2. Anticipatory Routing / SwiGLU Clamping 的理论机理待解。
  3. Agent 能力短板：Terminal Bench 输给 GPT-5.4；R&D 编码落后 Opus 4.6 Thinking（80% vs 67%）。
  4. 多模态缺席：V4 仍是纯文本，明确"正在做"。
  5. 更稀疏 embedding（conditional memory via scalable lookup）、更低延迟架构 是下一步探索。
10
For Practitioners

给从业者的五条启示

架构研究者

mHC（流形约束 + Sinkhorn-Knopp）是非常优雅的 residual 升级。CSA + HCA 的混合思路比单纯 MoBA/NSA 更系统，分层混合策略值得借鉴。

长上下文产品团队

异构 KV cache + on-disk storage 让多轮对话、agent 共享 system prompt、IDE 类工具成本降一个数量级。

后训练工程师

Specialist → OPD 流水线解决"多能力合并变弱"问题，对 RL 超参不敏感。GRM 进一步降低 reward engineering 成本。

训练稳定性

训 MoE 遇 loss spike？Anticipatory Routing 是可插拔急救包（仅 spike 激活），SwiGLU Clamping 近乎零成本。

开源生态

V4-Flash（284B / 13B 激活）是"可落地推理"的甜点，配 FP8 存储 + 磁盘 KV cache，部署成本明显低于 V3。
