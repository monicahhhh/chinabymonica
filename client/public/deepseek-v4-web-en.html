Positioning · Direction

Efficiency makes million-token context real

If V4 had to be summarized in a single word, it would be efficiency. Where competitors chased "stronger reasoning" and "bigger scale," V4 confronts the most fundamental bottleneck of Transformers — quadratic attention complexity — head-on.

Despite having more activated parameters, V4-Pro uses 3.7× less compute and 9.5× less KV cache than V3.2 at 1M tokens. V4-Flash pushes KV cache to roughly 1/14. These are not incremental optimizations; they are an architectural rewrite.

03
Four Core Innovations

Four engines of the efficiency revolution

3.1  Hybrid Attention · Interleaved CSA + HCA

The heart of the paper. V4 designs two complementary attention mechanisms and then interleaves them per layer, rather than picking one.

CSA · Compressed Sparse Attention — "compress first, then sparsify"

  • Compression stage: every m=4 tokens are collapsed into a single KV entry via a dual-stream Cₐ/Cᵦ formulation with overlapped windows, learnable positional biases, and a row-wise softmax over 2m weights. Not mean pooling — a learnable, position-aware KV summary.
  • Sparse stage: DSA applied on the compressed sequence via a Lightning Indexer (low-rank queries + ReLU similarity + top-k). Each query attends only to its top-k compressed blocks.
  • Sliding-window patch: parallel SWA branch (n_win=128) preserves strict local dependencies that compression + sparsity might lose.
  • Attention Sink: learnable per-head sink logit lets heads abstain (total attention < 1) — an old StreamingLLM trick that still pays off.

HCA · Heavily Compressed Attention — "extreme compression + dense attention"

If CSA is moderate compression (m=4), HCA is extreme: m′=128 — 128 tokens of KV collapsed into a single entry. The resulting sequence is already tiny, so HCA skips sparsification and runs full dense attention. Simpler (no indexer, no top-k) but dramatic compression.

The near-field view of CSA, plus the global glance of HCA — every layer sees the whole context.

Three-tier precision strategy

RoPE dims ………… BF16 (position info is precision-sensitive) KV (other dims) … FP8 (halves cache vs BF16) Indexer QK ……… FP4 (top-k doesn't need precision)

Combined with CSA/HCA length compression, total KV cache shrinks to ~2% of a BF16 GQA8 baseline. That's why million-token context becomes routine rather than aspirational.

3.2  mHC · Manifold-Constrained Hyper-Connections

Hyper-Connections expand the residual stream from ℝᵈ to ℝ^(n_hc × d) via three linear maps — more expressive, but numerically unstable when stacked deep. V4's fix: constrain B_l to the Birkhoff polytope of doubly stochastic matrices.

B_l ∈ M := { M ∈ ℝ^{n×n} | M·1_n = 1_n, 1_n^T·M = 1_n^T, M ≥ 0 }
  • ‖B_l‖₂ ≤ 1 → the map is non-expansive; signals cannot amplify in either direction → numerical stability.
  • M is closed under multiplication → stability composes across deep stacks.
  • Projection via 20 Sinkhorn-Knopp iterations (exp → alternating row/column normalization).
  • A_l, C_l clamped via sigmoid to non-negative bounded values — no signal cancellation.
Why it matters

RESIDUAL UPGRADE FOR DEEP NETS

mHC gives anyone training 100+ layer models a theoretically grounded, operationally usable residual upgrade. Engineering cost — added activation memory + pipeline comm — is hidden by fused kernels, tensor-level recomputation, and DualPipe 1F1B overlap, landing at just 6.7% wall-clock overhead.

3.3  Muon · A Widespread Replacement for AdamW

V4 migrates the vast majority of parameters from AdamW to Muon, keeping AdamW only for embeddings, the LM head, RMSNorm weights, and mHC's static biases and gates.

accumulate momentum M_t
Nesterov correction + hybrid Newton–Schulz
approximate orthogonalization (σ → 1)
rescale update RMS by √max(n,m)·γ
share hyperparameters with AdamW · WD + update
Hybrid Newton-Schulz

First 8 steps use aggressive coefficients (3.4445, −4.7750, 2.0315) for fast convergence. Final 2 steps switch to conservative (2, −1.5, 0.5), pinning singular values precisely at 1. Fast and stable.

Reconciling ZeRO with Muon

Muon needs full gradient matrices — conflicts with ZeRO. V4 uses a knapsack-based bucket assignment + flattened MoE distribution + BF16 stochastic-rounded gradient sync + all-to-all + FP32 local reduction. <10% memory overhead, comm volume halved.

Payoff: faster convergence + greater stability. QK-Clip is no longer needed — the attention architecture lets RMSNorm directly control Q/K scales.

3.4  DeepSeekMoE Continuation + Hash Routing Early Layers

  • Affinity function: Sigmoid → Sqrt(Softplus(·)) — empirically more stable.
  • Removed cap on routing target nodes; parallelism strategy redesigned to compensate.
  • First 3 MoE layers use Hash Routing — experts deterministically assigned by hashing token ID. Avoids shallow-layer instability before routing has learned anything meaningful.
  • MTP depth = 1, unchanged from V3.
04
Infrastructure

Making 1M training actually possible

Arguably the most impressive part of the paper — DeepSeek showcasing its full-stack model-engineering capability.

Fine-grained MoE overlap

MoE split into 3 expert waves; each wave's comm hidden behind the previous wave's compute. Theoretical speedup 1.92× (vs Comet 1.42×), tolerates lower interconnect bandwidth.

TileLang

Custom domain-specific language balancing dev productivity and runtime — replaces hand-written CUDA.

Batch-Invariant deterministic kernels

Bitwise reproducibility across training, inference, and RL rollout — critical for on-policy algorithms.

FP4 quantization-aware training

MoE expert weights + indexer QK use FP4. RL rollout uses real FP4 weights — sampling-time behavior exactly matches deployment.

Tensor-level checkpointing

Annotate a few tensors; framework traces the graph via TorchFX, finds the minimal subgraph for recomputation, dedups shared storage. Zero overhead, full convenience.

Contextual Parallelism (two-stage)

① Each rank sends its last m uncompressed KV entries to rank i+1. ② Rank i+1 compresses with local data → all-gather → fused select-and-pad kernel reassembles.

Heterogeneous KV cache × on-disk storage

Full Caching / Periodic Checkpointing / Zero Caching — operators trade storage for compute. Key enabler of commercial million-token deployment, especially for shared-prefix reuse (multi-turn, agent frameworks).

DSec production-grade sandbox

Rust on 3FS. Four substrates (Function Call / Container / microVM / fullVM) under one Python SDK. Hundreds of thousands of concurrent sandboxes per cluster; preemption-safe resume via trajectory logs.

05
Pre-training & Stability

Two practical stability recipes

Data & training setup

  • 32T+ tokens of curated multilingual corpus: filters for batch-generated content, upweighted long documents, expanded long-tail languages, agentic data injected mid-training.
  • Sequence length: 4K → 16K → 64K → 1M. First 1T tokens dense-attention warmup; sparse attention introduced at 64K.
  • Batch peak: 75.5M (Flash) / 94.4M (Pro). Peak LR: 2.7e-4 / 2.0e-4, cosine decay to 1/10.
  • Auxiliary-loss-free balancing + small sequence-wise balance loss; MTP weight 0.3 → 0.1 during decay.

Two fixes for loss spikes

The authors candidly report frequent spikes during trillion-parameter MoE training. Root cause: MoE-layer outliers amplified by the routing mechanism.

Anticipatory Routing

Decouple feature computation from routing decisions: at step t, use θ_t for features but θ_{t−Δt} for routing indices (precomputed, cached). Breaks the "feature → outlier → routing → more outliers" cycle.

Auto-detect spikes, activate temporarily, deactivate on recovery → negligible overall overhead.

SwiGLU Clamping

Clamp linear component to [−10, 10]; cap gate at 10. Simple, nearly free, empirically eliminates outliers.

Candor

"IT WORKS, WE DON'T FULLY KNOW WHY"

The paper openly states the theoretical mechanism of both techniques remains insufficiently understood, and invites community research. That candor is refreshing in today's over-polished technical reports.

Post-Training Paradigm

Specialist × On-Policy Distillation

The biggest methodological shift in V4: OPD replaces mixed RL in the merging phase.

Base Model
SFT on high-quality, domain-specific data
GRPO RL per domain · math / code / agent / instruction-following
Multi-Teacher On-Policy Distillation (reverse KL)
Unified Final Model
  • Truly on-policy — no distillation distribution shift
  • Avoids cross-task interference endemic to multi-task RL merging
  • Training loop batches teacher inference & shares KV cache → huge compute savings

Generative Reward Model (GRM)

V4 abandons scalar reward models entirely. The actor network itself serves as the reward model, trained on rubric-guided RL data. Judging skill and generation skill are optimized jointly. Human annotation budget drops dramatically.

Three reasoning effort modes

More post-training details

  • Tool-Call · XML with |DSML| special tokens: replaces JSON; eliminates escaping failures; integrates cleanly with interleaved <think>.
  • Interleaved Thinking: V4 preserves the entire reasoning history across turns in tool-calling scenarios (even across user-message boundaries). This is what makes 1M context useful for long-horizon agents.
  • Quick Instruction: auxiliary tasks (web search trigger, intent recognition) handled by appending dedicated special tokens, reusing existing KV cache — significantly lowers TTFT.
  • WAL + persistent KV cache: preempted rollouts resume at the token level. Regenerating from scratch would introduce a length bias — shorter outputs survive interruption more often — biasing the model toward shorter responses. Hence the insistence on true resumption.

Four Deep Takeaways

Four deeper reads

① From "parameter efficiency" to "system efficiency"

V3's banner was activated parameters (MoE + MLA). V4's banner is context energy efficiency — FLOPs per token, KV cache per token. In the reasoning-model era, long CoTs / long tool traces / long document analyses shift compute from the parameter axis to the sequence axis. V4 is a direct response.

② The 3–6-month frontier gap has changed nature

On hard reasoning (HMMT, IMO, Codeforces, LiveCodeBench), V4-Pro-Max is effectively on par or ahead of Gemini-3.1-Pro and GPT-5.4. On knowledge-intensive benchmarks (SimpleQA, MMLU-Pro), V4 still trails considerably.

The remaining gap is not reasoning ability — it's data scale × knowledge breadth.

③ On-Policy Distillation may become the dominant post-training paradigm

Train N specialists separately; have the student learn teachers' distributions via reverse KL on its own trajectories. Combine with GRM, batch-invariant deterministic kernels, and asynchronous teacher scheduling, and you get a standardized, scalable post-training pipeline.

④ The infrastructure story matters as much as the architecture story

Chapter 3 (General Infrastructures) and §5.2 (RL Infrastructure) together span nearly 20 pages — TileLang DSL, FP4 QAT, 3FS, DSec sandbox, tensor-level checkpointing. These are what make architectural creativity actually run on 100K-GPU clusters × trillion-parameter models × million-token contexts. The infrastructure is the moat.

09
Limitations

Authors' own shortcomings

  1. Architectural complexity is high — many "validated but not minimally necessary" components retained to reduce risk. Future work: principled distillation.
  2. Anticipatory Routing & SwiGLU Clamping lack theoretical grounding.
  3. Agent gap remains: Terminal Bench trails GPT-5.4; R&D coding behind Opus 4.6 Thinking (80% vs 67%).
  4. Text-only — multimodal listed as in-progress.
  5. Future directions: sparser embeddings (conditional memory via scalable lookup), lower-latency architectures, deeper long-horizon agent iteration.

For Practitioners

Five implications for builders

Architecture researchers

mHC (manifold constraint + Sinkhorn-Knopp) is an elegant residual upgrade. The CSA + HCA layer-interleaved hybrid is more systematic than MoBA/NSA alone — worth porting.

Long-context product teams

Study heterogeneous KV cache + on-disk storage. Multi-turn chat, agent frameworks with shared prompts, IDE tools — costs can drop an order of magnitude.

Post-training engineers

Specialist → OPD solves "merged model = weakest link" without fragile RL hyperparameter tuning. GRM further reduces reward-engineering cost.

Training stability

Training MoE and seeing loss spikes? Anticipatory Routing is a pluggable rescue kit (activate only during spike). SwiGLU Clamping is near-zero cost.

Open-source ecosystem

V4-Flash (284B / 13B activated) is a deployment sweet spot. With FP8 KV cache + on-disk storage, inference costs meaningfully lower than V3.
